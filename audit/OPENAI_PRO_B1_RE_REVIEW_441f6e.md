# 3. 跨页面阻断：完整页面在动画前泄露结论

## P1-B1-INTEGRATION-001

三个动画单独打开时已经比上一版好，但学生不是从 iframe 文件开始阅读，而是从 `bx1.md` 上方正文开始阅读。

### B1-12

在 iframe 之前，正文已经明确告诉学生：

- 共速时距离最远；
- 共速不是相遇；
- 本模型最终一定追上；
- `Δx-t` 与 `v-t` 的解释。

因此动画到共速前提出“这是相遇还是距离极值”的预测时，答案已经被页面正文给出。

### B1-18

在“默认引导演示”之前已经出现完整公式卡：

```text
0≤f_s≤f_s,max
f_s,max=μ_sN
f_k=μ_kN
```

情境段也已直接给出“比较接触面间相对运动或趋势”的最终判断方法。动画虽然隐藏公式，页面却没有隐藏。

### B1-24

主问题刚提出，下一段立即显示：

```text
F合=ma
```

静态图及图注也在 iframe 前直接写出 `F合=ma`。这使动画内部的“最后才揭示公式”失去意义。

### 修改要求

三个节点统一改为：

```text
锚定问题
→ 默认引导演示 iframe
→ 观察证据与反馈
→ 因果解释
→ 规范定义/公式
→ 边界与高考迁移
```

课标信息可放在问题之后或折叠；静态总结图和公式卡必须移到动画证据之后。

### 验收

- 学生按页面从上到下阅读，在作出第一项预测前看不到该预测的规范答案；
- 不仅独立动画，完整 Docsify 页面也通过教学路径测试；
- 新增页面集成测试，而不是只测试 `file://anim/...html`。

## 5.2 P1-B1-18-PHYS-001：`F=0` 初态出现错误的相对运动趋势

初始状态为：

```text
F=0
f=0
```

但画面文字仍写：

> 若去掉摩擦：箱底有向右滑动趋势

没有外力时不存在该向右相对运动趋势。

### 修复

```text
F=0：
尚无水平相对运动趋势，摩擦力为 0。

0<F≤f_s,max：

```

### 自动测试

增加断言：

```text
初态 F=0 时不得出现“向右滑动趋势”。
```

## 5.3 P1-B1-18-FLOW-002：分步模式跳过“临界—滑动”证据

自动模式中，`F` 从 `20 N` 增加至 `32 N`，会经过：

```text
静止
→ f_s,max
→ 滑动
→ f_k
```

但分步模式执行：

```js
state.F=32;
finish();
```

`finish()` 立即把情境切为传送带，学生看不到 `F=32 N` 时箱子已滑动、摩擦切换为 `f_k` 的画面。

这意味着系统偏好减少动态效果的学生，反而失去最关键的临界证据。

### 修复

分步模式至少拆成：

```text
步骤 1：F=20 N，静摩擦按需
步骤 2：F=25 N，达到最大静摩擦
步骤 3：F=32 N，已经滑动，显示 f_k
步骤 4：进入传送带方向迁移
```

每一步都必须停留，学生点击“下一证据”后继续。

- 方向预测的“验证”只展示传送带结果。
