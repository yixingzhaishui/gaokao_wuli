# 3. 发布阻断问题

## P0-B1-12：追及相遇的确定物理错误仍在正文

### 当前正文

`bx1.md` 的 B1-12 情境仍写：

> 如果后车加速度足够大，它最终会追上前车；如果加速度太小、前车太快，后车永远追不上。

### 为什么错误

该页明确使用：

- 前车做匀速直线运动；
- 后车保持正的恒加速度；
- 时间、道路长度和速度均不设上限。

在这个模型里，后车位移包含正的二次项：

```text
x后 = v10 t + 1/2 a t²
```

只要 `a>0` 并持续作用，二次项最终超过前车的一次项，因此后车最终一定追上。加速度小只会使追上时间更晚，不能导致“永远追不上”。

“永远追不上”只有增加以下条件之一才可能成立：

- 加速只持续有限时间；
- 后车存在最高速度；
- 道路或观察时间有限；
- 后车之后转为不高于前车速度的匀速；
- 其他动力学约束。

### 正确改写

建议第一段改为：

> 在本页的理想模型中，前车保持匀速、后车持续以正的恒加速度追赶，且时间和道路不设上限，因此后车最终一定会追上；加速度越小，追上越晚。若题目增加限速、有限加速时间或道路终点，才需要重新判断能否追上。

同时把：

> “速度相等”是追及问题的临界点

收紧为：

> 速度相等表示两车距离的变化率为零，因此对应距离极值；它是最大值还是最小值，要结合相对速度在该时刻前后的符号判断。相遇条件始终是位置相同。

### Codex Action

1. 修正 `bx1.md` B1-12 第一段及所有同义表述。
2. 检查 `anim/bx1/pursuit.html` 默认演示终点。
3. 默认演示必须运行到真正相遇，不得停在共速后但相遇前。
4. 在共速前暂停，先问“此刻是相遇，还是距离极值？”
5. 增加有限加速或限速模式时，必须明确它是另一个模型。

### 验收

- 全仓不再出现“恒正加速度太小所以永远追不上”的无条件说法；
- 默认参数演示能运行至 `x后=x前`；
- 共速与相遇分别显示；
- `Δx=0` 和 `v后=v前` 不被混为同一条件。

## P0-B1-24-PHYSICS：粗糙面模式静摩擦读数错误

### 当前实现

`anim/bx1/newton-second.html` 使用：

```js
const f = state.mu * state.m * G;
const net = Math.max(0, state.F - f);
```

并始终把 `f=μmg` 作为摩擦读数和箭头显示。

### 确定错误

例如默认：

```text
F = 6 N
m = 3 kg
μ = 0.32
g = 10 m/s²
```

代码得到：

```text
μmg = 9.6 N
F合 = max(0, 6-9.6) = 0
```

物体若静止，实际静摩擦应为 `6 N`，与外力平衡，而不是 `9.6 N`。当前画面会显示一个大于外力的摩擦力，同时又显示合力为零，受力图内部矛盾。

### 修复方案

最清晰方案：

1. 牛顿第二定律默认控制变量实验只用光滑面，令阻力为零；
2. 粗糙面作为演示后的拓展；
3. 拓展中分开：
   - `μ_s`
   - `μ_k`
4. 若物体初始静止：
   ```text
   F ≤ μ_s mg → f_s = F，F合 = 0
   F > μ_s mg → 开始滑动，f_k = μ_k mg，F合 = F - f_k
   ```
5. 若页面只想展示恒定阻力，不要把它命名为静摩擦或用同一个 `μ` 描述。

### 验收

- 任一静止状态下，摩擦力不会大于造成相对运动趋势的外力需求；
- 静摩擦、最大静摩擦和滑动摩擦清楚分开；
- 粗糙面受力箭头、数值、合力和加速度一致；
- 增加一条语义测试：`F<μ_smg` 时应有 `f_s=F`。
