# 《四川高考物理》教学设计规范（Teaching Style Guide）

> 版本：1.0  
> 生效日期：2026-07-17  
> 适用范围：全部知识点、模型专题、实验专题、高考能力专题、STSE 与拓展页面  
> 配套规范：[动画设计规范](ANIMATION_STYLE_GUIDE.md)、[内容编辑模板](CONTENT_TEMPLATES.md)、[数学符号规范](MATH_SYMBOLS.md)、[题目来源与发布政策](SOURCE_POLICY.md)

---

## 1. 使命与产品定位

本项目不是习题册，也不是公式查询站。

每一页都要帮助学生完成：

```text
看见现象
→ 感受到变化
→ 产生“为什么”
→ 从证据中形成概念或规律
→ 理解数学表达
→ 知道适用边界
→ 能预测新情境
→ 最后迁移到高考
```

高考是理解后的迁移与验证，不是页面起点，也不能用题量替代教学。

学生完成一页后，应至少能做到一项：

- 预测条件改变后的结果；
- 解释现象的因果链；
- 说明公式为什么成立、什么时候失效；
- 区分高频混淆概念；
- 把动画、图像、读数和公式对应起来；
- 看出高考题包装的是哪一个已理解的物理过程。

若学生只能复述定义或公式，不能预测、解释或判断边界，该页不得标记为 `content approved`。

---

## 2. 学生时间与认知约束

默认读者是时间有限的高中学生，不是长时间自由探索的研究者。

| 页面类型 | 首轮建议时长 | 目标 |
|---|---:|---|
| 基础概念 | 4～7 分钟 | 建立一个概念、一个反例 |
| 核心规律 | 7～12 分钟 | 看证据、懂关系、会判断边界 |
| 综合模型 | 10～18 分钟 | 建立对象—过程—约束—方程链 |
| 实验专题 | 12～25 分钟 | 看装置、会操作、会处理数据与误差 |
| STSE/拓展 | 5～10 分钟 | 理解机制、证据和限制 |

首屏应在 30 秒内让学生知道：

1. 本页解决什么问题；
2. 应观察什么；
3. 学完后能解释或预测什么。

普通知识点以教师设计好的演示路径为主。学生不拖动多个参数，也应能看到完整物理过程。自由探索是可选层；完整虚拟实验只用于实验专题。

## 3. 全站统一教学原则

### 3.1 先产生问题，再给概念

不得把定义作为页面第一教学动作。

不推荐：

```text
质点是有质量而没有大小和形状的理想化模型。
```

推荐：

```text
为什么地图上可以把一列很长的高铁画成一个点，
但研究它完全通过隧道时又不能这样做？
```

概念应作为解决问题的工具出现，而不是要求学生先接受的结论。

### 3.2 先展示证据，再总结规律

规律页尽量经历：

```text
控制条件
→ 观察变化
→ 比较数据或图像
→ 学生先说出关系
→ 页面给出规范结论
```

“先显示公式，再调滑块证明”只能用于复习模式，不能作为首次教学主线。

### 3.3 “为什么”必须解释因果

以下不算解释：

- 因为它符合定义；
- 因为公式就是这样；
- 因为这是某某定律；
- 把结论换一种说法再重复。

合格解释应回答：

- 哪个量变了；
- 什么作用、相互作用或约束导致变化；
- 哪些条件保持不变；
- 为什么不是学生容易猜测的另一种结果；
- 公式各部分如何对应画面、读数或图像；
- 去掉哪个条件后结论会改变。

### 3.4 每页一个主问题

一页可以有多个知识要素，但必须只有一个主问题。其他内容全部服务于主问题，不能平均铺开。

例如 B1-06 的主问题应是：

> 加速度描述速度怎样变化，为什么它的正负不能直接等同于加速或减速？

### 3.5 理解结果必须可检验

页面结束前，学生应能：

- 在未给答案前预测曲线或运动结果；
- 指出一个错误解释错在哪里；
- 说明临界条件来自哪个边界；
- 解释同一对象为何在不同问题中采用不同模型。

## 4. 通用页面结构

不同页面采用不同模板，但都应形成以下闭环。栏目可合并，不得机械堆标题。

### 4.1 第一屏：锚定问题

第一屏包含：

- 一个真实现象、认知冲突或具体任务；
- 一句明确主问题；
- 学完后可完成的预测或解释。

第一屏不得出现：

- 大段课标元数据；
- 多个公式；
- 大量题型列表；
- 未解释的术语堆叠；
- “本节要求记住……”式命令。

课标定位可折叠或放到锚定问题之后。

### 4.2 引导演示

默认演示应有：

- 开始状态；
- 连续变化；
- 关键节点；
- 一次核心条件对比；
- 规律收束。

关键处提出预测。画面、读数、矢量、图像和公式必须同步。学生不操作也能完成主线学习。

具体规则见 `ANIMATION_STYLE_GUIDE.md`。

### 4.3 观察证据

观察任务必须可核对。

不推荐：

> 观察加速度如何变化。

推荐：

> 保持质量不变，把合力调为原来的 2 倍。比较加速度读数和 `a-F` 图上的点：加速度是否也变为 2 倍？

建议数量：

- 基础概念：1～2 个；
- 核心规律：2～4 个；
- 综合模型：按阶段设置，首轮不超过 5 个；
- 实验专题：按完整流程设置。

### 4.4 因果解释

推荐顺序：

```text
发生了什么
→ 为什么发生
→ 哪个量描述它
→ 这个量如何测量或计算
```

解释要主动回应学生高频疑问，而不是只解释标题。

### 4.5 概念与规律

定义必须：

- 指出研究对象；
- 说明物理意义；
- 区分相近概念；
- 避免循环定义；
- 避免把特例写成普遍结论。

规律必须注明：

- 控制条件；
- 方向或符号约定；
- 适用范围；
- 极端、临界或失效情形。

### 4.6 数学表达

公式是压缩物理关系、进行预测的工具，不是页面目标。

公式首次出现时紧邻说明：

- 公式解决什么问题；
- 每个符号对应什么画面量；
- 单位；
- 矢量或标量属性；
- 正方向；
- 模型假设和适用条件；
- 哪个量保持不变时才可谈正比或反比。

无定量公式的概念页不得强行插入“公式首次使用卡”。应使用“模型边界卡”“概念判据卡”，或不设置卡片。

高中范围内可从实验、图像或已学规律得到的关系，应展示关键来源。大学层级公式或微积分推导标为 `extension`，并明确“不作为核心记忆要求”。

### 4.7 反例与边界

每页至少提供一个能推翻机械记忆的反例或边界，例如：

- 速度大但加速度为零；
- 速度为零但加速度不为零；
- 摩擦力可以是动力；
- 完全失重不等于没有重力；
- 接触不一定有弹力；
- 绳不能提供负拉力；
- 静摩擦不总等于 `μN`。

反例必须解释为什么不矛盾。

### 4.8 高考迁移

高考部分放在理解形成之后，回答：

1. 为什么这里容易设陷阱；
2. 学生最常混淆哪两个量、条件或过程；
3. 题目怎样用图像、实验或真实情境包装；
4. 改变哪个条件后熟悉结论会失效。

普通知识点建议：

- 1 个快速预测；
- 1 个错误理解辨析；
- 1 个可追溯高考迁移任务。

不以重复计算题数量评价页面质量。题目发布必须遵守 `SOURCE_POLICY.md`。

### 4.9 学习收束

结尾使用：

```text
我看见了什么？
我为什么能这样解释？
条件改变后，我能预测什么？
```

可增加一句核心认识，但不能写成脱离条件的口诀。

## 5. 不同内容类型的专用要求

### 5.1 概念页

目标：建立可辨析的物理意义。

必须有：

- 需要该概念的真实问题；
- 正例和反例；
- 与相近概念的对比；
- 一次情境判断；
- 模型或概念边界。

不得用大量计算题掩盖概念不清。

### 5.2 规律页

目标：从证据形成变量关系，并能用于预测。

必须有：

- 控制变量演示或数据；
- 图像或表格；
- 正比、反比成立的控制条件；
- 公式的物理解释；
- 极端、临界或失效情形。

### 5.3 模型页

目标：把复杂情境简化为可求解结构。

统一流程：

```text
研究对象
→ 过程和阶段
→ 模型假设
→ 约束关系
→ 真实受力或能量关系
→ 选择规律
→ 分阶段求解
→ 边界与可行性检查
```

禁止：

```text
看到关键词
→ 选模型名
→ 套固定公式
```

模型识别必须依据对象、约束和过程。

### 5.4 实验页

目标：让学生理解证据如何支持结论，并会完成高考实验任务。

必须有：

- 探究问题；
- 自变量、因变量、控制变量；
- 器材规格、量程和安全；
- 规范操作顺序；
- 原始数据；
- 作图、拟合或计算；
- 偶然误差、系统误差和过失；
- 误差方向；
- 异常数据处理；
- 改进方案；
- 结论的证据边界。

### 5.5 STSE 与拓展页

必须区分：

- 课程核心结论；
- 技术应用；
- 估算模型；
- 生命周期或系统边界；
- 不确定性和现实权衡。

拓展内容明确标注 `extension`，不计入核心覆盖，也不得让学生误以为必须记忆大学层级公式。

## 6. 教学语言规范

语言应准确、具体、可视化、有因果，不追求口号感。

推荐：

> 最高点速度暂时为零，但重力仍向下，所以加速度仍为 `g`，物体随后向下运动。

不推荐：

> 最高点不是静止，要特别注意。

写作要求：

- 一个段落完成一个教学动作；
- 重要结论前先给证据或问题；
- 避免连续多层“因为—所以—因此—从而”；
- 避免“显然”“容易看出”等空泛词；
- 不用“记住”“背下来”代替解释；
- 不使用“压轴”“真题风格”等无证据标签；
- 不用大学术语替代高中解释；
- 全站遵守 `MATH_SYMBOLS.md`；
- 同一物理量不得无说明更换符号。

## 7. 信息密度与页长

建议范围用于识别失控页面，不作为机械字数门禁。

| 页面类型 | 建议正文长度（不含代码、题库） |
|---|---:|
| 基础概念 | 900～1600 中文字 |
| 核心规律 | 1500～2600 中文字 |
| 综合模型 | 2200～4200 中文字，可按过程折叠 |
| 实验专题 | 2500～5000 中文字，可按流程折叠 |
| STSE/拓展 | 1000～2200 中文字 |

优先删除或折叠：

- 重复定义；
- 同一结论的同义复述；
- 与主问题无关的大学拓展；
- 多道只换数字的题；
- 动画已清楚呈现、正文又逐帧重复的内容。

必须保留和增强：

- 学生真正会问的“为什么”；
- 因果解释；
- 反例；
- 公式条件；
- 图像与实物对应；
- 高考中的条件变化与模型失效。

## 8. 状态与审核工作流

### 8.1 状态分离

以下状态独立，不得相互自动升级：

```yaml
curriculum_map:
physics_content:
pedagogy_review:
formula_review:
source_review:
interaction_qa:
mobile_qa:
publish:
```

交互测试通过不等于内容审定通过。

### 8.2 内容通过条件

页面只有同时满足以下条件，才可将 `physics_content` 和 `pedagogy_review` 标为通过：

- 物理结论、方向、量纲和边界正确；
- 第一屏提出明确问题；
- 默认演示能完成主线教学；
- 有预测—观察—解释闭环；
- “为什么”是因果解释；
- 公式条件紧邻公式；
- 至少一个反例或失效条件；
- 高考迁移建立在理解之后；
- 未核验题目没有进入学生端；
- 手机端可完成主线学习；
- 内容长度和信息密度适合学生。

### 8.3 存量修改原则

新规范生效后，旧页面首先复审，不默认推倒重写。

修改顺序：

1. 保留正确物理内容和已有高质量动画资产；
2. 重排教学顺序；
3. 增加默认演示和预测节点；
4. 删除重复与机械模板；
5. 修正条件、符号和错误解释；
6. 最后才考虑重建页面或动画。

只有存在核心物理错误、教学模型根本错误，或原结构无法支持因果理解时，才判定 `REWRITE`。

## 10. 页面终审清单

### 物理

- [ ] 研究对象和过程明确；
- [ ] 物理量、方向、单位准确；
- [ ] 特例没有写成普遍规律；
- [ ] 公式条件紧邻首次出现位置；
- [ ] 反例和边界正确；
- [ ] 图像、读数、动画与公式一致。

### 教学

- [ ] 第一屏让学生知道为什么要学；
- [ ] 先有现象和问题，再有定义与公式；
- [ ] 默认演示无需复杂操作；
- [ ] 至少完成一次预测—观察—解释；
- [ ] 因果解释不是同义重复；
- [ ] 页面只有一个主问题；
- [ ] 学生形成可检验的预测能力。

### 高考

- [ ] 高考内容放在理解之后；
- [ ] 解释高频混淆和设陷方式；
- [ ] 题目来源符合发布政策；
- [ ] 题量没有替代教学；
- [ ] 难度标注有客观依据。

### 出版与体验

- [ ] 标题层级、术语和符号一致；
- [ ] 无机械模板残留；
- [ ] 无未解释的大学拓展；
- [ ] 手机端可读可操作；
- [ ] 加载失败时保持安全状态；
- [ ] 状态标签真实反映审定进度。

---

## 11. 规范优先级

若文件冲突，按以下顺序执行：

1. 物理正确性与课程标准；
2. 本教学设计规范；
3. `ANIMATION_STYLE_GUIDE.md`；
4. `CONTENT_TEMPLATES.md`；
5. 页面既有结构和历史写法。

旧模板不得成为拒绝改进教学顺序的理由。
