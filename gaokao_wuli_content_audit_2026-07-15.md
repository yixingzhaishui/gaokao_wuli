## 0. 结论先行

### 0.1 发布判定

**当前版本不通过“正式高考教辅 / 高三学生可直接依赖”的发布验收。**

更准确的定位应是：

> **交互功能较完整、内容体系较丰富的内部测试版 / 教学原型。**

在 P0 问题全部关闭前，不应继续使用下列对外口径：

- “覆盖四川新高考物理全部内容”；
- “全部完成”；
- “已核验题库”已达到可直接替代教辅题库的程度；
- 统一的 `done / 96 / 98 分` 等同于物理内容已由高三教师审定。

### 0.2 本轮确认的核心判断

现有工作对**动画可操作性、移动端跳转、真实控件测试**投入较充分，这是明显优势；但“控件能动”不等于“教辅可用”。本轮静态总审与高风险抽查已经确认：

1. **课程标准映射有结构性错位**：光电效应被列入选择性必修1，同时又列入选择性必修3；必修3应有的磁场、电磁感应基础内容主要被挪到选择性必修2。
2. **“官方学生必做实验”口径错误**：课标为 21 个，当前页面把 22 个列为课标必做，侧栏又把全部 26 个统称为“官方必做实验”。
3. **存在确定的数值答案错误**：B3-16 闭合电路例题把支路功率重复乘以电阻，正确值约为 `0.735 W`，页面写成约 `2.94 W`。
4. **来源治理文件损坏**：`SOURCE_POLICY.md` 与 `SOURCE_VERIFICATION_REPORT.md` 不是有效文档，而是本地 `sed` 找不到文件的报错文本。
5. **旧例题的隔离机制是“运行时、失败放行”**：数据加载失败时，正文中的无可靠来源例题可能重新显示；静态打印、离线、脚本失败等情境下也缺少可靠封锁。
6. **进度口径互相冲突**：交互审核“全部通过”、README 总完成率 77.4%、必修1 全部 review、教学说明仍写多个批次“待开始”，没有区分“交互完成”和“内容审定”。
7. **多处表述省略适用条件**，会诱导高三学生机械套公式，例如图像斜率、I-U 斜率、向心临界、端电压、激光发散角、全反射临界角等。
8. **大量“高考真题改编 / 四川适应性演练 / 高考真题风格”没有题号、年份、原卷页码和改编记录**，不符合教辅书的来源与编辑责任标准。
9. **公式与符号存在可见排版错误**，如 `ν 0`、`p_{1}\ \mathrm{V}1/T_{1}`，说明现有检查未覆盖学生最终看到的数学表达。

### 0.3 审核边界

本轮完成的是：

- 仓库治理文件、课程目录、进度系统、题库发布逻辑的总审；
- B1、B2、B3、X1、X3、实验、能力专题、模型专题、题库的高风险抽查；
- 若干公式、数值例题和教学表述的独立复核；
- 形成全站后续逐页审定的统一标准与机器验收项。

本轮**不冒充**“已经把 226 个节点、全部例题、全部动画逐字逐式人工审完”。恰恰因为抽查已经发现确定答案错误和系统性口径问题，现有 `done / 98` 不能外推为其余内容正确；必须执行本报告后续的全量复算与教师审签。

## 1. 审核依据

### 1.1 官方依据

1. 《普通高中物理课程标准（2017年版2020年修订）》
   <https://www.esph.com.cn/docs/2025-04/b8fb91c002624b2e90ebfb1095f23a32.pdf>
2. 四川省教育考试院《关于做好我省2026年普通高校招生工作的通知》
   <https://www.sceea.cn/Html/202604/Newsdetail_4767.html>

四川省 2026 年通知明确强调：

- 加强教考衔接；
- 考查基础知识、基本技能、基本方法；
- 使用科技前沿、项目式、探究式真实情境；
- 考查关键能力、学科素养和思维品质。

因此，本项目不能只做到“知识点有文字、动画可点击”，还必须做到：

- 物理事实正确；
- 条件与边界完整；
- 模型建构合理；
- 证据、图像、公式和结论能互相对应；
- 学生能迁移到真实情境，而不是背口诀；
- 题目来源、答案和难度标注可信。

### 1.2 教辅书质量标准

正式教辅至少应满足：

1. **知识归属准确**：模块、课标条目、必学/拓展不混淆。
2. **概念表述严谨**：不把特例写成普遍规律，不省略关键条件。
3. **公式完整**：变量定义、方向、单位、适用条件、极端情形齐全。
4. **例题可追溯**：真题必须能定位到原卷；原创题必须明确标为原创。
5. **答案可复核**：数值、符号、单位、有效数字和结论均独立复算。
6. **难度标注真实**：不能把基础辨析题包装成“压轴”。
7. **教学梯度合理**：概念—单规律—多过程—真实情境逐步提升。
8. **状态真实**：编辑、物理审定、来源核验、交互测试必须分开记录。
9. **静态也安全**：脚本失败、打印、离线时不暴露未审核内容。
10. **可由教师使用**：能备课、能讲解、能布置、能诊断，不只是学生浏览页面。

## P0-CALC-001　已确认闭合电路例题数值答案错误；全站答案必须重新独立复算

**定位**

- `bx3.md`
- `B3-16 闭合电路欧姆定律`
- “例题 2（高考真题改编，中档）”

**原题数据**

- `ε = 3 V`
- `r = 1 Ω`
- `R₁ = 2 Ω`
- `R₂ = 4 Ω`
- 两电阻并联

**页面前半部分计算**

```text
R并 = 4/3 Ω
I总 = 9/7 A
U = 12/7 V
I₂ = 3/7 A
```

上述均正确。

**正确功率**

```text
P₂ = I₂²R₂
   = (3/7)² × 4
   = 36/49 W
   ≈ 0.735 W
```

**当前页面错误**

页面写成：

```text
(3/7)²×4 = 36/49×4 ≈ 2.94 W
```

即已经把 `×4` 算入 `36/49` 后又重复乘了一次。

**风险**

- 这是确定答案错误，不是措辞争议；
- 证明当前 `done / 98` 不能作为“答案已经审定”的证据；
- 若学生按解析训练，会形成错误结果；
- 同类模板生成内容可能存在更多重复代入、单位和有效数字错误。

**Codex 修改要求**

1. 立即把该题答案改为 `36/49 W ≈ 0.735 W`。
2. 查明该题所谓“高考真题改编”的来源；没有完整来源则改为“原创训练题”。
3. 对全站每一道数值题执行**独立复算**，不得只检查最终字符串是否存在。
4. 为结构化题目增加可机检字段，例如：

```json
{
  "expected": {"value": 0.7346938776, "unit": "W"},
  "tolerance": 0.001,
  "check_expression": "(3/7)^2*4"
}
```

5. 对符号题增加等价式检查或人工双签。
6. 生成 `audit/content-solution-recalculation.md`，记录：
   - 题目 ID；
   - 原答案；
   - 独立推导；
   - 复核者；
   - 是否修正。

**验收条件**

- 该题修正；
- 所有发布题都有复算记录；
- 至少两种独立路径抽查高风险题；
- 数值、单位、有效数字、方向和选项字母一致；
- 任何一题复算失败都会阻断发布。

## P0-SRC-001　正文仍含大量不可追溯“真题改编”，不符合教辅来源标准

**定位示例**

- `bx1.md`：`教材改编`、`高考常见模型改编`、`高考综合模型改编`
- `bx2.md`：`四川适应性演练`、`高考真题改编`、`高考真题风格改编`
- `bx3.md`：`高考真题改编`
- `xb1.md`：`2019 改编`、`2020 改编`、`高考真题风格`
- 其他模块同类标签

**问题**

这些标签通常缺少：

- 年份；
- 卷名；
- 题号；
- 原卷页码；
- 原题图；
- 原答案来源；
- 改了什么；
- 改编后是否仍保持原结论；
- 原卷可访问链接。

“高考真题风格”更不是来源，只是一种营销式描述。

**教辅标准**

允许三类写法：

1. **真题原题**
   明确年份、卷名、题号、来源、核验记录。
2. **据真题改编**
   除以上信息外，还要写改编项与改编目的。
3. **原创训练**
   明确写“原创”，不得借“真题风格”制造权威感。

**Codex 修改要求**

1. 全仓搜索：

```text
高考真题改编
高考真题风格
四川适应性演练
高考常见模型改编
2019 改编
2020 改编
教材改编
```

2. 每一处做三选一：
   - 补齐完整来源；
   - 改为“据××题改编”并写改编记录；
   - 改为“原创训练题”。
3. 没有来源的题不得标为“真题”“适应性演练”。
4. 难度和来源必须是独立字段，不能用“高考风格”同时暗示二者。
5. 将正文例题统一由结构化数据生成，避免 Markdown 自由文本漏审。

**验收条件**

- 仓库中不存在无元数据的“真题/适应性/联考”标签；

## P0-RUNTIME-001　未核验例题隔离为“失败放行”，静态与异常情境不安全

**定位**

- `index.html`
- `hideUnverifiedExercises()`
- `isPublishableProblem()`

**当前逻辑**

页面先把未核验例题写在 Markdown 正文里，再由浏览器运行 JavaScript：

1. 拉取题库；
2. 找到“例题/训练题”标题；
3. 用已核验题替换；
4. 隐藏后续旧内容。

但在 `fetch` 失败时，`catch` 中没有执行安全隐藏，正文会继续保留。

**风险**

以下情境可能重新暴露旧例题：

- 网络或 JSON 请求失败；
- GitHub Pages 缓存不一致；
- JavaScript 被禁用；
- 打印、离线或静态抓取；
- Docsify 插件异常；
- 数据格式错误导致函数提前退出。

这与“未核验内容下架”的声明冲突。

**Codex 修改要求**

优先方案：

1. 从学生端 Markdown 源文件中物理删除未核验例题；
2. 移入 `data/quarantine/`；
3. 构建时根据题库生成安全的静态例题块；
4. 生成失败则构建失败，不部署。

最低可接受方案：

- 页面初始状态默认不含旧题；
- 数据成功后才插入已核验题；
- 失败时显示“题库暂不可用”，绝不显示旧题。

同时：

- `errors.md`、`papers.md` 等纯 JS 挂载页应提供有意义的无脚本回退；
- 增加关闭 JavaScript、阻断 JSON 请求的端到端测试；
- 测试打印 DOM 中是否含隔离题文本。

**验收条件**

- 禁用 JS 时，不出现任何未核验例题；
- 题库请求返回 404/500/无效 JSON 时，不出现旧题；
- 静态 HTML/打印内容只含已核验题或明确的不可用提示；
- 构建失败不得发布旧内容。

## P0-QBANK-001　题库“可发布”检查只验证字段存在，尚未证明来源与答案可靠

**定位**

- `data/verified-problems.json`
- `data/verified-diagnostic-2024cee.json`
- `index.html#isPublishableProblem`
- `papers.md`

**当前逻辑的优点**

已经检查：

- `source_kind`
- `source_verified`
- `exam`
- `question_no`
- `source_urls`
- `answer`
- `solution`
- 图题是否有图

这是一个好的起点。

**但仍缺少**

- 来源网站是否为考试院、官方出版物或仅为第三方题库；
- 原卷文件哈希与页码；
- 答案是来自官方答案、权威解析还是人工推断；
- 两个链接是否其实复制自同一份错误文本；
- 题干、选项、图、答案是否逐字一致；
- 重绘图是否改变关键几何关系；
- 解析是否独立复算；
- 来源链接失效后的证据留存；
- 同题不同版本冲突处理。

当前样例中存在 21 世纪教育网、第三方 PDF 镜像等链接。第三方镜像可以作为辅助证据，但仅有第三方链接不能自动证明“官方原卷来源”。

**Codex 修改要求**

增加来源分级：

| 等级 | 来源 | 发布规则 |
|---|---|---|
| A | 教育考试院、教育部、正式官方试卷/答案、正式出版社可核文件 | 可直接作为主证据 |
| B | 权威媒体原卷扫描、学校/教研机构正式发布、可靠镜像且可与 A/另一独立来源交叉核对 | 可发布，但注明镜像 |
| C | 普通题库站、博客、论坛、二次录入 | 只能辅助定位，不能单独支撑“已核验” |
| D | 无原卷、来源不明、AI生成文本 | 隔离 |

新增字段：

```json
{
  "primary_source_url": "...",
  "primary_source_tier": "A",
  "scan_page": 3,
  "file_sha256": "...",
  "answer_source_url": "...",
  "answer_source_tier": "A",
  "stem_checked_by": "...",
  "answer_checked_by": "...",
  "solution_recalculated_by": "...",
  "figure_fidelity_checked_by": "...",
  "verification_status": "double_checked"
}
```

**验收条件**

- 每个发布题至少有 A，或“两个相互独立的 B + 明确无 A 的说明”；
- C 类链接不能作为唯一证据；
- 图题有原图页码与重绘核验记录；
- 答案与解析由不同步骤核验；
- `isPublishableProblem` 不再只检查布尔字段，而检查来源等级和双签。

## P0-MATH-001　学生可见公式存在损坏，需建立数学表达质量门禁

**定位示例**

- `xb3.md`：`ν 0`、`\nu 0`
- `xb3.md`：`p_{1}\ \mathrm{V}1/T_{1}=p_{2}\ \mathrm{V}2/T_{2}`
- 多处纯文本 `Ek`、`fmax`、`P_max`、上下标和变量混用
- 页面中同一量交替使用 `E`、`ε`、`W0`、`W_{0}`

**风险**

- `ν 0`会被理解为两个量相乘或变量加数字；
- `\mathrm{V}1`是排版错误，不是 `V_1`；
- 公式即使“能看懂”，也不符合高三板书、答卷和教辅规范；
- 搜索、屏幕阅读和后续自动检查均受影响。

**Codex 修改要求**

1. 建立统一符号表：
   - 截止频率 `\nu_0`
   - 最大初动能 `E_{k,\max}`
   - 电动势 `\mathcal{E}` 或全站统一 `E`
   - 体积 `V_1,V_2`
   - 最大静摩擦 `f_{s,\max}`
2. 增加 Markdown/KaTeX 检查：
   - 未闭合公式；
   - 裸上下标；
   - `ν 0` 等可疑模式；
   - `\mathrm{V}1`；
   - 单位与变量混排。
3. 在真实浏览器中截图抽查所有公式密集页。
4. 公式旁必须有变量定义与条件，不能只有公式串。

**验收条件**

- 全仓公式 linter 通过；
- 高风险模式搜索结果为零；
- KaTeX 控制台无错误；
- 物理教师抽查公式、符号和单位通过。

| **选择性必修1** | 波、光交互有价值 | 光电效应错误归属；偏振、激光、光纤表述需修 | **红色阻断** |
| **选择性必修2** | 现有交互审核证据较充分 | 需与必修3重新切分基础/深化；全量审安培力、洛伦兹力、感应、交流电条件 | 映射修复后专项复审 |
| **选择性必修3** | 热学、近代物理内容较全 | 与 X1 重复；符号损坏；部分大学公式/定量拓展未分层 | 修复后复审 |
| **实验专题** | 有“装置—操作—数据—误差”意识 | 21项官方口径错误；实验名称与探究性质不一致 | **红色阻断** |
| **高考能力专题** | 有利于二轮复习 | 口诀省略条件，容易机械化；斜率、面积、电路选择有误导风险 | 改为判定流程 |
| **模型专题** | 模型诊断思路有价值 | 关键词匹配强于约束分析，公式特例被当一般式 | 强化“对象—约束—过程—边界” |
| **题库** | 已有结构化数据和隔离意识 | 治理文档损坏、来源等级不足、运行时失败放行 | **红色阻断** |
| **交互与移动端** | 现有审计材料显示完成度较高 | 交互正确不代表内容正确；需把两种状态拆开 | 保留成果，不能替代内容审定 |

# 6. 高三教师视角的统一改写标准

## 6.1 每个概念/规律页

建议结构：

1. **课标定位**：条款、必学/拓展、常见题型。
2. **学习目标**：用“能判断 / 能解释 / 能计算 / 能迁移”表达。
3. **前置诊断**：2—3 个问题检查学生已有概念。
4. **真实现象或证据**。
5. **建立模型**：研究对象、忽略因素、坐标与正方向。
6. **规律形成**：从实验、图像或推理得到，不直接空降公式。
7. **公式卡**：
   - 公式；
   - 符号；
   - 单位；
   - 矢量/标量；
   - 适用条件；
   - 极限/临界情形。
8. **典型例题**：来源清楚，步骤体现评分点。
9. **变式训练**：只改变一个关键条件。
10. **易错诊断**：错误原因而不只是“正确答案”。
11. **章末迁移**：与后续模型、实验、真实情境连接。

## 6.2 每个模型页

统一流程：

```text
识别研究对象
→ 画过程与状态
→ 写约束关系
→ 画真实受力或能量流
→ 选择定律
→ 分阶段列式
→ 检查边界和可行域
→ 回答题目要求
```

禁止只教：

```text
看到关键词 → 套模型名字 → 套固定公式
```

## 6.3 每个实验页

必须覆盖：

- 实验目的；
- 可观察的因变量；
- 自变量和控制变量；
- 原理；
- 器材规格与安全；
- 规范操作顺序；
- 原始数据表；
- 数据处理；
- 作图与拟合；
- 偶然误差、系统误差、过失；
- 误差方向；
- 改进方案；
- 异常数据处理；
- 结论的适用范围；
- 高考答题语言。

## 6.4 每道题的解析

至少包括：

1. 题目信号；
2. 研究对象和过程；
3. 核心条件；
4. 方程及依据；
5. 计算；
6. 单位与有效数字；
7. 结论；
8. 易错分支；
9. 来源或原创声明。

# 7. 难度分级规则

禁止只用“基础 / 中档 / 压轴”主观标注。建议采用四级：

| 等级 | 认知与过程特征 | 示例 |
|---|---|---|
| **L1 基础识记/直接判断** | 单一概念，无转换 | 单位、方向、定义判断 |
| **L2 单规律应用** | 一个模型、一步或两步计算 | 牛顿第二定律直接计算 |
| **L3 综合迁移** | 两个以上规律、多表征或分过程 | 图像 + 动力学、实验数据处理 |
| **L4 高综合/开放** | 多过程、隐含约束、真实情境、论证或方案评价 | 复杂综合题、探究设计 |

“压轴”只能作为某份具体试卷中的位置描述。没有具体试卷，不应给原创基础题贴“压轴”。

scripts/check-static-fallback.js

模拟：

- 题库 JSON 404；

- 不出现空白核心页；

检查：

- README、syllabus、页面状态一致；
- 章节标题层级；
- 不存在“partial 但全部 done”；
- 不存在“待开始但统计 100%”；

# 15. 本轮审核中的正向评价

本报告结论严格，但项目已有值得保留的基础：

1. **交互设计方向正确**：大量页面让学生拖动参数、看图像和读数联动，而不是只播放装饰动画。
2. **观察任务意识较强**：多数页面不是只给公式，而是要求学生观察变量变化。
3. **适用条件和易错点已经进入模板**：虽然部分条件仍不严谨，但结构方向正确。
4. **移动端和真实控件测试较认真**：这类工程质量是传统教辅缺少的优势。
5. **已经意识到来源问题并尝试隔离旧题**：需要把“运行时补救”升级为“构建时可信治理”。
6. **模型、实验、能力专题均有独立入口**：重建权威映射后，可形成很好的复习系统。
7. **不应推倒重来**：正确做法是保留交互资产，重做课标映射、内容审定、题目来源和状态治理。

# 16. 总结

当前版本最主要的问题不是“内容太少”，而是：

> **内容很多，但权威目录、物理审定、题目来源和完成状态还没有形成出版级闭环。**

因此，下一阶段不应继续单纯增加节点、动画和题量，而应优先完成：

```text
课标一一映射
→ P0错误清零
→ 全量答案复算
→ 来源双重核验
→ 条件化改写
→ 教师审签
→ 再发布
```

在上述闭环建立后，这个项目有机会成为比传统静态教辅更强的交互式高考物理学习系统；在闭环建立前，它只能被诚实地标为“审校中的教学原型”。
